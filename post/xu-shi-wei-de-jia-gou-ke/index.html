2023-06-25 4 min read

汇编语言的诞生

在变成语言出现之前，程序员需要将要表达的执行直连翻译成二进制的比特数据，然后将其刻录到存储上。【刻录还涉及到硬件的电气操作】

为了解决编程效率的问题，汇编语言【和解释它的编译器】诞生了。汇编语言的编译器将汇编语言写的程序编译成为CPU指令序列，将其保存在存储设备上。

汇编语言做了如下的效率优化：

  • 用文本符号表达机器指令。例如 add 表示加法运算，比CPU的加法二进制指令好记忆。
  • 用文本符号表达要操作的内存地址。例如 用变量名指向存储文本的内存地址，这样我们就不用关心内存的物理地址。
  • 用文本符号表示调用的函数地址。
  • 用文本符号表示要跳转的目标地址。跳转的目标地址用文本符号表示有助于程序逻辑的表达，而不是让人把精力放到具体的指令跳转地址上。

汇编语言非常接近计算机的CPU指令，基本上一条汇编指令对应一条CPU指令，但是汇编将人们从物理硬件地址中解脱出来，以便更专注于程序逻辑的表达。

汇编语言的出现要早于操作系统。操作系统的核心目标是软件治理，只有在计算机需要管理很多的任务时，才需要操作系统。

编程语言的进化

编程范式的进化

1. 过程式

过程式就是以一条条的命令的方式，让计算机按我们的意愿来执行。
计算机的机器语言本身就是一条条指令构成的，本身也是过程式的，所以过程式最为常见。
代表语言有：C、C++。

2. 函数式

函数式本质上是过程式编程的一种约束。它最核心的主张就是变量不可变，函数尽可能没有副作用。

副作用是什么？
举个例子：内部有IO行为的函数，就有副作用
既然变量不可变，函数没有副作用，自然人们犯错的机会也就更少，代码质量就会更高。代表语言有：Haskell、Erlang。

3. 面向对象

面向对象在过程式的基础上，引入了对象（类）和对象方法（类成员函数），它主张尽可能把方法（其实就是过程）归纳到合适的对象（类）上，不主张全局函数（过程）。代表语言有：Java、Go、C#、C++.
面向对象还有一个至关重要的概念，就是接口。通过接口，我们可以很容易的实现过程式编程中很费劲才能实现的一个能力：多态。

4. Java：继承和组合

  • 继承：动物和猫、狗。
  • 组合：汽车和引擎、车轮。
    Java 同时支持继承和多态，Go 放弃继承、全面强化组合能力。

不同的编程范式并不是互斥的。例如Java是纯正的面向对象语言，它反对全局过程。但Go语言是多范式的。
确切的说，Go是面向连接的编程语言，所谓面向连接就是朴素的组合思想。研究连接，就是研究代码和代码之间如何组合。

5. Go：消息传递

消息传递约束了进程的行为，是进程与进程间的连接契约。消息传递是多核背景下流行起来的一种编程思想，其核心主张是：尽可能用消息传递来取代共享内存，从而避免显式的锁，提高性能，降低编程负担。
Go语言不仅仅是提供了语言内建的消息传递机制channel，同时它的消息传递是类型安全的。

其他进化

从语言的执行器的行为看，出现了这样三类分类的语言：

  • 编译的目标文件为可执行程序，例如：C、C++、Go。【可执行文件不跨平台】
  • 生成跨平台的虚拟机字节码，有独立的执行器(虚拟机)执行字节码文件。例如：Java、Erlang。
  • 直接解释执行，例如：JavaScript。
